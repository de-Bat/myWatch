# Android (Capacitor) Design

**Date:** 2026-06-02  
**Status:** Approved

## Overview

Wrap the existing myWatch Next.js web app as a native Android APK using Capacitor. Approach: replace NextAuth with a direct JWT auth layer, switch to static export, add Capacitor shell.

## Constraints & Decisions

- Email/password auth only for v1 (no Google/Apple native OAuth)
- Self-hosted server — user enters API URL in Profile settings
- Offline data via Dexie (IndexedDB); no service worker in Android build
- Web deployment also migrates from standalone → static export (no server required)

## Section 1: Auth Layer Replacement

### Remove
- `apps/web/src/auth.ts`
- `apps/web/src/app/api/auth/[...nextauth]/route.ts`
- `apps/web/src/app/api/health/route.ts`
- `next-auth` npm dependency

### Add
- `apps/web/src/lib/auth-store.ts` — JWT read/write abstraction over `localStorage`
- `apps/web/src/hooks/useAuth.ts` — replaces `useSession`. Interface:
  ```ts
  { user: { id, email, name } | null, token: string | null, signIn(email, password): Promise<void>, signOut(): void, isLoading: boolean }
  ```
- Auth pages (`/auth/login`, `/auth/register`, `/auth/forgot-password`, `/auth/reset-password`) call `apiClient` directly

### Migration
All `useSession` callsites swap to `useAuth`. Only `session?.user` and `session?.apiToken` are read — mechanical replacement. Files affected: `apps/web/src/app/page.tsx`, `apps/web/src/components/MediaPanel.tsx`, `apps/web/src/app/media/[type]/[id]/page.tsx`.

## Section 2: Next.js Static Export + Capacitor Config

### `apps/web/next.config.mjs` changes
- `output: 'standalone'` → `output: 'export'`
- Add `trailingSlash: true`
- Remove `outputFileTracingRoot` (standalone-only option)
- Serwist disabled when `DISABLE_SW=true` env var is set

### New: `apps/web/capacitor.config.ts`
```ts
import { CapacitorConfig } from '@capacitor/cli'

const config: CapacitorConfig = {
  appId: 'com.mywatch.app',
  appName: 'myWatch',
  webDir: 'out',
  server: { androidScheme: 'https' },
  plugins: {
    Preferences: {},
  },
}

export default config
```

### New packages (`apps/web`)
- `@capacitor/core`
- `@capacitor/android`
- `@capacitor/preferences`

### Build pipeline
```sh
# In apps/web:
DISABLE_SW=true pnpm build        # outputs to out/
npx cap sync android               # copies out/ into android/ project
npx cap open android               # opens Android Studio for APK build
```

### Android project
`apps/android/` — generated by `cap add android`, checked into repo. Contains `AndroidManifest.xml`, Gradle config, app icons.

## Section 3: Android-Specific Considerations

### Routing
`trailingSlash: true` causes each route to emit its own `index.html`. Capacitor serves `out/` as local files — deep links work without a server.

### Back button
Hook into Capacitor `App` plugin `backButton` event. Navigate back via Next.js router; exit app if at root.

### Network / API URL
`apiClient` reads base URL from user settings (already implemented). No change needed — user enters self-hosted server URL in Profile settings.

### Cleartext traffic
Add `android:usesCleartextTraffic="true"` to `AndroidManifest.xml` to support HTTP self-hosted servers (common for local Jellyfin installs).

### Icons & splash
Generate from single source image via `@capacitor/assets`.

## Out of Scope (v1)
- Google/Apple Sign-In
- Push notifications
- Deep link intent filters
- Play Store signing / release pipeline
